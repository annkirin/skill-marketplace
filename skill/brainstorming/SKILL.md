# Brainstorming Skill

每次开始任务前，先按以下流程进行头脑风暴：

## 第一步：明确问题

- 任务的核心目标是什么？
- 约束条件有哪些？（时间、技术栈、资源等）
- 成功的标准是什么？

## 第二步：发散思考

从以下角度自由联想，不加评判：

| 角度 | 问题 |
|------|------|
| 功能 | 需要实现什么能力？用户能做什么？ |
| 体验 | 用户会怎么用？流畅度如何？ |
| 技术 | 用什么方案？有哪些技术选型？ |
| 边界 | 异常情况怎么处理？极限情况？ |
| 扩展 | 未来可能怎么演化？预留什么接口？ |
| 风险 | 哪里可能出问题？最坏情况？ |

## 第三步：归类整理

1. **亲和图法**：把相似的想法归为一组
2. **优先级排序**：按 重要性/紧急度 四象限分类
3. **关系连线**：找出想法之间的依赖和关联

## 第四步：聚焦决策

- 从每组中选出 1-3 个最佳方案
- 明确下一步行动项（Action Items）
- 确定第一个实现的 MVP 范围

## 输出格式

完成任务后输出以下结构：

```
## 头脑风暴结果

### 核心目标
...

### 关键方案
- [方案1] （优先级：高/中/低）
- [方案2] （优先级：高/中/低）

### 行动项
- [ ] 第一步：...
- [ ] 第二步：...

### 待验证
- 需要进一步确认的地方...
```
